Location Inventory Management (Perform Inventory)

Use Case ID LI-05 Version 1.0
Use Case Name Perform Inventory Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse, Warehouse Worker, Store Manager, Store
Other Actor(s) Delivery Driver
Stakeholders
  • Inventory Control
Description This use case enables users to conduct physical inventory counts at specified locations. The process includes counting and verifying quantities on hand, updating records, and identifying any discrepancies between physical and system-recorded stock levels.
Pre Condition(s) User has authorization to perform inventory counts.
Location data and existing inventory records are up-to-date.
Triggers User selects "Perform Inventory" in the inventory management system to initiate a physical count.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. User initiates inventory count. System displays inventory items for the selected location
2. User enters counted quantities. System captures counts and compares them to current records.
3. User confirms completion. System updates records and flags any discrepancies.
4. System generates inventory report. Report is generated, detailing count results and flagged items.

Alternate Flow

Actor Action System/Result, Info required
1. User pauses inventory count. System saves progress and allows resumption later.
2. Discrepancies identified. System prompts user to verify count for flagged items.
3. User cancels inventory action. System discards all changes and exits the counting interface.
Conclusion Inventory count is completed, and records are updated. Discrepancies are flagged for follow-up.
Post Condition(s) Inventory records are refreshed to reflect accurate quantities, and any discrepancies are logged for further investigation.

Additional Information

Business Rules
  • Only authorized users can perform inventory counts.
  • All discrepancies must be reviewed and resolved by authorized personnel.
Special Requirements Real-time access to the inventory system during the count.
Ability to save and resume inventory sessions as needed.
Constraints and Specifications Counts must be verified by a secondary user for audit purposes, if possible.
Technology and Data System-integrated scanning tools (if available) to expedite counting.
Access to inventory records in real-time for immediate updates.
Frequency Quarterly for regular audits, and as needed based on inventory levels or specific location requirements.
Assumptions Locations have designated staff and resources to conduct accurate counts.
Notes / Open Issues Determine if specific protocols for high-value items are required during inventory.
