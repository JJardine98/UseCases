USE CASE NARRATIVE

Order Management (Create Store Order)

Use Case ID OM-01 Version 1.0
Use Case Name Create Store Order Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Store Manager
Other Actor(s) Warehouse Manager
Stakeholders
  • Inventory Management
  • Sales
  • Procurement
Description This use case allows the Store Manager to create an order to replenish stock. The system automatically includes items that fall below the minimum reorder level, ensuring consistent inventory levels at retail locations.
Pre Condition(s) The system contains data on current stock levels and minimum reorder quantities.
The Store Manager has the necessary permissions to place orders.
Triggers A request from the Store Manager to submit a new order.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1) Store Manager selects "Create Store Order" on the dashboard System opens order creation interface and displays low-stock items
2) Store Manager reviews and adjusts order quantities as needed System updates order details dynamically
3) Store Manager confirms and submits the order System logs the order and sends it to the warehouse
4) Store Manager receives confirmation Order confirmation is shown to the Store Manager

Alternate Flow

Actor Action System/Result, Info required
Store Manager includes additional items not flagged as low System adds items to order and updates quantities
Store Manager sets a delivery preference (e.g., standard or priority) System logs the priority and adjusts scheduling, if needed

Conclusion

The Store Manager successfully submits an order for stock replenishment, and the order is queued for processing by the warehouse.

Post Condition(s)

The system updates inventory records to reflect the pending order and notifies the warehouse for fulfillment.

Additional Information

Business Rules
  • Only authorized Store Managers can create and submit store orders.
  • Automatic addition of low-stock items must comply with the pre-defined reorder thresholds.
Special Requirements The system should provide a user-friendly order interface, with options to adjust quantities and prioritize certain items as needed.
Constraints and Specifications Access is limited to authorized users, and order adjustments can only be made before submission.
Technology and Data Real-time synchronization with the inventory system for accurate stock level tracking.
Frequency Typically conducted weekly or as needed based on stock levels.
Assumptions Store Managers have access to the system and receive notifications of low-stock items regularly.
Notes / Open Issues Clarify any restrictions on changing order details once submitted to the warehouse.
