USE CASE NARRATIVE

Order Management (Create Emergency Order)

Use Case ID OM-02 Version 1.0
Use Case Name Create Emergency Order Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Store Manager
Other Actor(s) Warehouse Worker, Warehouse Manager
Stakeholders
  • Inventory Management
  • Sales Operations
Description This use case allows the Store Manager to submit an emergency order for essential inventory items that are in immediate need. Emergency orders are limited in quantity due to urgency and supply constraints and are prioritized for fulfillment.
Pre Condition(s) The Store Manager is logged into the system with permissions to create emergency orders.
System has current inventory data and pre-set limits for emergency order quantities.
Triggers An item or set of items in the store's inventory falls below critical levels, necessitating an emergency restock.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
Store Manager navigates to the "Create Emergency Order" screen System displays the emergency order form with item entry fields
Store Manager enters the items and quantities needed System validates items and ensures quantities are within set limits
Store Manager submits the emergency order System logs the order, assigns an emergency priority, and notifies the warehouse for processing

Alternate Flow

Actor Action System/Result, Info required
Store Manager attempts to exceed the emergency order quantity limit System displays an error message and restricts quantity to set limits

Conclusion

The emergency order is successfully submitted and flagged for priority processing by the warehouse.

Post Condition(s)

The system updates the store’s pending order list with the emergency order, tagged with high priority for immediate handling.

Additional Information

Business Rules
  • Emergency orders are limited to a maximum number of items or quantity per order, depending on set thresholds.
  • Only authorized personnel (Store Manager) can submit emergency orders.
Special Requirements
  • System should allow notification alerts to Warehouse for any emergency order submissions.
  • Emergency orders must be visible in a prioritized queue in the warehouse order processing system.
Constraints and Specifications The system should enforce quantity limits to ensure emergency orders remain feasible for quick fulfillment.
Technology and Data Real-time integration with inventory and order processing systems for immediate updates.
Frequency Emergency orders are placed as needed, typically infrequent but prioritized when they occur.
Assumptions Inventory data is accurate, and emergency thresholds are predefined and relevant.
Notes / Open Issues Discuss the possibility of automated notifications or alerts to upper management for each emergency order submission.
