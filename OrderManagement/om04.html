USE CASE NARRATIVE

Order Management (Process Store Order)

Use Case ID OM-04 Version 1.0
Use Case Name Process Store Order Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse Worker
Other Actor(s) Store Manager
Stakeholders
  • Warehouse Operations
  • Store Management
Description This use case describes the steps for processing a store order in the warehouse. It involves checking item availability, assigning quantities to fulfill the order, recording backordered items if needed, and updating the order status for tracking.
Pre Condition(s) A store order has been created and is ready for processing in the warehouse system.
The Warehouse Worker has access rights to process store orders.
Triggers A request to process a store order from the system or directly from the Store Manager.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
Warehouse Worker selects the “Process Store Order” option System displays pending orders to be processed
Actor selects an order to process System loads the selected order, including items and requested quantities
Actor checks item availability for each item System displays current inventory levels and flags items that are low in stock
Actor assigns available quantities to fulfill the order System updates quantities assigned and generates a pick list for the order
Actor records any backordered items System flags items as backordered and adds them to the backorder list
Actor confirms processing completion System updates the order status to “Processed” and prepares it for fulfillment

Alternate Flow

Actor Action System/Result, Info required
Actor finds items unavailable for immediate fulfillment System prompts actor to record unavailable items as backordered and updates the order status accordingly

Conclusion

The store order is processed, items are assigned, and the order is updated to reflect quantities and any backorders.

Post Condition(s)

The order is moved to the “Ready for Fulfillment” status, and any backorders are flagged for restocking.

Additional Information

Business Rules
  • Orders should be processed in the sequence they are received to maintain fairness and efficiency.
  • Backordered items must be restocked and prioritized for future orders once available.
Special Requirements
  • System should track item availability in real-time to streamline order processing.
  • Automatic alerts for low stock on high-demand items.
Constraints and Specifications Order processing should occur within a set time frame to meet delivery schedules.
Technology and Data Inventory and order management data should be synchronized in real-time across warehouse and store systems.
Frequency Store orders are typically processed daily, as received from various store locations.
Assumptions The inventory system is updated with accurate, real-time stock levels.
Notes / Open Issues Consider implementing automatic backorder creation for items consistently in high demand.
