USE CASE NARRATIVE

Supplier and Backorder Management (View Backorders)

Use Case ID SBM-04 Version 1.0
Use Case Name View Backorders Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse Manager
Other Actor(s) Store Manager
Stakeholders
  • Procurement
  • Inventory Management
Description This use case enables warehouse and store managers to view all backorders, helping them track items that are pending delivery due to stock shortages. Users can view current backorders, filter by specific dates, and monitor fulfillment statuses.
Pre Condition(s)
  • The system contains data on backordered items due to previous inventory shortages.
  • Access rights are granted to authorized personnel to view backorders.
Triggers A request from the warehouse or store manager to view backorders.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Manager selects "View Backorders" from the dashboard. System displays a list of active backorders.
2. Manager applies filters (e.g., date range, supplier, item type). System updates the list based on filters.
3. Manager selects a specific backorder for details. System shows detailed information, including expected delivery dates and quantities.
4. Manager notes or exports backorder information as needed. System generates a report, if required.

Alternate Flow

Actor Action System/Result, Info required
1. Manager searches for historical backorders. System displays closed backorders within the date range.
2. Manager reviews unresolved backorders. System flags these for further action, if needed.

Conclusion

The manager successfully views backorder information, with the option to drill down into details for specific items.

Post Condition(s)

The system records any actions taken (e.g., export, filter adjustments) and maintains an accurate, up-to-date backorder list.

Additional Information

Business Rules
  • Only authorized users can access backorder data.
  • Backorder records must be kept for a specified period for reporting and audit purposes.
Special Requirements Data filtering options must be comprehensive to allow for tailored reporting (e.g., by date, item type).
Constraints and Specifications System should limit access to only active and authorized users to ensure data confidentiality.
Technology and Data Real-time integration with the inventory system to reflect backorder status updates.
Frequency As needed, especially when items are low in stock or shipments are delayed.
Assumptions Warehouse and store managers have regular access to a workstation or device to view backorder information.
Notes / Open Issues Define specific access restrictions for viewing historical versus active backorders.
