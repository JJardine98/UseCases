USE CASE NARRATIVE

Supplier and Backorder Management (Add Items to Backorder)

Use Case ID SBM-05 Version 1.0
Use Case Name Add Items to Backorder Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse Worker
Other Actor(s) Warehouse Manager
Stakeholders
  • Procurement
  • Inventory Management
Description This use case allows the Warehouse Worker to create backorders for items that are currently unavailable in stock but are needed to fulfill store orders. This ensures that demand is logged and can be fulfilled when stock is available.
Pre Condition(s) The system is updated with current inventory levels and allows for backordering.
The Warehouse Worker has access to store order details indicating which items are out of stock.
Triggers A store order includes items that are out of stock, prompting a backorder.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Warehouse Worker selects "Add to Backorder" on the store order page System displays out-of-stock items for that order
2. Warehouse Worker confirms items to be backordered System generates a backorder entry with item details
3. Warehouse Worker confirms the creation of the backorder System assigns a backorder ID and updates the backorder list
4. Confirmation of backorder creation is displayed System displays confirmation of the backorder creation

Alternate Flow

Actor Action System/Result, Info required
1. Warehouse Worker sets a priority level for backordered items System prioritizes fulfillment when stock arrives
2. Warehouse Worker modifies quantities or adds notes for specific items System logs changes and updates the backorder entry

Conclusion

Items are successfully added to a backorder list, ready for processing when stock becomes available.

Post Condition(s)

The system logs the backorder and updates inventory needs, notifying procurement to restock these items as necessary.

Additional Information

Business Rules
  • Only authorized users can create and manage backorders.
  • Backorders should prioritize items based on order type (e.g., emergency or regular).
Special Requirements Backorder entries should be editable if additional items are later added to the store order.
Constraints and Specifications The system must track the dates of backorders and quantities per item for inventory planning.
Technology and Data Integration with the inventory and procurement systems to automatically trigger restocking.
Frequency Backorders are created as needed when items are out of stock.
Assumptions Procurement will address backordered items promptly to avoid delays.
Notes / Open Issues Consider adding automated notifications for backordered items when stock is replenished.
