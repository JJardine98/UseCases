USE CASE NARRATIVE

Supplier and Backorder Management (Add Items to Backorder)

Use Case ID SBM-05 Version 1.0
Use Case Name Add Items to Backorder Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Source Business Requirement
Primary Actor Warehouse Manager
Other Actor(s) Supplier, Admin
Stakeholders
  • Procurement
  • Finance
  • Inventory Management
Description This use case describes how a warehouse manager adds items to a backorder list to manage inventory shortages effectively.
Pre Condition(s) Items must be identified as out of stock and backorder must be allowed in the system.
Triggers The warehouse manager identifies an item that needs to be backordered.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Warehouse Manager selects "Add to Backorder". System prompts for item details and quantity.
2. Warehouse Manager enters item details. System validates item information.
3. Warehouse Manager confirms addition to backorder. System updates the backorder list and notifies relevant stakeholders.

Conclusion

The items are successfully added to the backorder list, ensuring that stock levels are managed effectively.
