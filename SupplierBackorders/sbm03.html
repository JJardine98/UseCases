USE CASE NARRATIVE

Supplier and Backorder Management (Receive Supplier Order)

Use Case ID SBM-03 Version 1.0
Use Case Name Receive Supplier Order Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Source Business Requirement
Primary Actor Warehouse Manager
Other Actor(s) Supplier, Admin
Stakeholders
  • Procurement
  • Finance
  • Inventory Management
Description This use case describes how a warehouse manager receives and processes a supplier order to update inventory levels.
Pre Condition(s) The supplier order must be confirmed and available in the system.
Triggers The warehouse manager receives the supplier order notification.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Warehouse Manager selects "Receive Order". System displays order details and prompts for confirmation.
2. Warehouse Manager confirms receipt of the order. System updates inventory levels and order status.

Conclusion

The supplier order is successfully received, and inventory is updated accordingly.
