Supplier and Backorder Management (Create Supplier Order)

Source Business Requirement
Primary Actor Warehouse Manager
Other Actor(s) Supplier, Admin
Stakeholders
  • Procurement
  • Finance
  • Inventory Management
Description This use case describes how a warehouse manager initiates and submits a purchase order for items from a supplier to replenish stock. This process is critical for maintaining adequate inventory levels and avoiding stockouts.
Precondition(s)
  • Supplier must be registered in the system.
  • Inventory levels should reflect the need for replenishment based on reorder thresholds.
  • User has warehouse management permissions to create orders.
Trigger(s) Inventory levels reach reorder threshold, or the warehouse manager identifies the need for additional stock and initiates an order.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Warehouse Manager selects "Create Supplier Order". System displays form for creating a new order.
2. Manager chooses supplier. System loads supplier details and available items.
3. Manager selects items and quantities. System verifies item availability and calculates order cost.
4. Manager reviews order. System displays summary for confirmation.
5. Manager confirms and submits order. System generates purchase order and sends it to the supplier.
6. System updates inventory on order confirmation. Notifies warehouse and admin of the submitted order.

Alternate Flow

Actor Action System/Result, Info required
1. Manager exits before submitting. System discards unconfirmed order and exits to main menu.
2. System detects insufficient funds. Displays error, prevents order submission until resolved.
3. Manager modifies order details. System updates calculations and redisplays summary.
Conclusion The supplier order is successfully created, reviewed, and submitted, initiating the supplier's fulfillment process.
Post Condition(s) The system records the order, updates inventory projections, and notifies relevant personnel.

Additional Information

Business Rules
  • Only authorized personnel can create and submit supplier orders.
  • Orders should only be placed with registered and approved suppliers.
  • Quantity limits may apply based on financial or storage constraints.
Special Requirements Orders should automatically notify finance for approval if costs exceed a set threshold.
Constraints and Specifications Only in-stock items can be ordered, and orders exceeding financial limits require review.
Technology and Data Integrated inventory management and ordering system, secure server to transmit orders to suppliers.
Frequency Orders are created as needed based on inventory levels and reordering policies.
Assumptions Suppliers can fulfill orders within a timeframe that meets warehouse restocking needs.
Notes / Open Issues Clarify if real-time inventory updates are needed once the supplier confirms an order.
