USE CASE NARRATIVE

Location Inventory Management (CRUD Locations)

Use Case ID LI-01 Version 1.0
Use Case Name CRUD Locations (Admin) Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Admin
Other Actor(s) None
Stakeholders
  • IT Department
  • Management
Description This use case allows the Admin to create, read, update, and delete (CRUD) records of physical store and warehouse locations in the system, ensuring accurate inventory and logistics data.
Pre Condition(s) Admin is logged in and authenticated.
System has a table or storage structure for location data.
Triggers Admin accesses the Locations Management module.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Admin selects "Create Location" System displays input fields for new location data
2. Admin enters location details (e.g., name, address) System validates and saves location in the database
3. Admin selects "View Location" System retrieves and displays location details
4. Admin selects "Edit Location" System displays editable fields for selected location
5. Admin updates location details System saves changes to the database
6. Admin selects "Delete Location" System prompts for confirmation, then deletes location if confirmed
Conclusion The Admin successfully creates, views, updates, or deletes location records in the system as required.
Post Condition(s) The system’s location data is updated to reflect any CRUD operations performed by the Admin.

Additional Information

Business Rules
  • Location names must be unique within the system.
  • Deletion of a location is only allowed if it has no active inventory or linked records.
Special Requirements Display confirmation dialog for deletions. System should log all CRUD actions for audit purposes.
Constraints and Specifications Operations should complete within acceptable timeframes (e.g., less than 2 seconds).
Technology and Data Database table for storing location information, logging actions.
Frequency As needed by the Admin, typically upon changes in the business’s physical locations.
Assumptions Admin has the authority to manage location data and follows company protocols.
Notes / Open Issues Implement validation to ensure no critical location data is accidentally deleted or modified.
