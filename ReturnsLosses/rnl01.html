USE CASE NARRATIVE

Returns & Loss Management (Process Return)

Use Case ID RNL-01 Version 1.0
Use Case Name Process Return Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse
Other Actor(s) Store Manager
Stakeholders
  • Store Operations
  • Inventory Management
Description This use case involves handling item returns to inventory. If the returned item is in usable condition, it is added back to inventory; otherwise, it is disposed of or flagged as non-sellable.
Pre Condition(s) Items have been identified as returned by a store or customer.
Triggers Warehouse or store manager initiates a return process for items.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Store Manager selects "Process Return" System prompts for return item details, including condition.
2. Store Manager enters item ID, quantity, and condition. System verifies item data and condition (usable or non-usable).
3. Store Manager submits return request. System adjusts inventory levels accordingly.
4. Warehouse verifies item condition upon receiving. System confirms return entry and updates the status as received..

Alternate Flow

Actor Action System/Result, Info required
1. Item details incomplete or condition unclear. System requests additional information to proceed with the return.
Conclusion Returned items are processed, and inventory levels are updated based on item usability.
Post Condition(s) Inventory is updated, and records indicate whether the item is restocked or marked as non-sellable.

Additional Information

Business Rules
  • Usable returns are restocked; non-usable returns are disposed of per company policy.
Special Requirements Returned items must be verified upon receipt for accurate inventory adjustments.
Constraints and Specifications System must support return classifications (usable, non-usable).
Technology and Data Integration with inventory system for real-time stock updates.
Frequency As needed, based on return frequency.
Assumptions Return details are accurately entered by store or warehouse staff.
Notes / Open Issues None
