USE CASE NARRATIVE

Location Inventory Management (Move Inventory)

Use Case ID LI-04 Version 1.0
Use Case Name Move Inventory Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse, Warehouse Worker, Store Manager, Store
Other Actor(s) Delivery Driver
Stakeholders
  • Inventory Control
  • Management
Description This use case allows authorized users to transfer inventory within or between warehouse and store locations. This includes preparing items for transport, updating location records, and confirming arrival at the destination.
Pre Condition(s) User has authorization to initiate inventory movement.
Sufficient stock exists at the source location for the transfer.
Triggers User selects the "Move Inventory" option from the inventory management interface, specifying source and destination locations.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. User selects source and destination. System displays available inventory at the source location.
2. User specifies items and quantity. System verifies availability and validates input.
3. User confirms transfer. System updates inventory levels at both source and destination locations.
4. System assigns tracking for the move. System generates tracking details and notifies the destination for receiving.

Alternate Flow

Actor Action System/Result, Info required
1. User cancels transfer action. System discards changes and returns to the previous screen.
2. Insufficient inventory at source. System notifies user of unavailable quantity and requests a lower amount.
Conclusion Inventory items are successfully transferred to the specified destination, with stock levels updated in the system.
Post Condition(s) The destination location’s inventory reflects the new items, and the source inventory is reduced accordingly.

Additional Information

Business Rules
  • User must have authorization to move inventory..
  • Source inventory must meet or exceed the quantity requested for transfer.
Special Requirements System should provide confirmation notifications upon successful transfer..
Movement details should be timestamped for tracking purposes.
Constraints and Specifications Inventory movements must be recorded in real-time for accuracy.
Technology and Data Real-time inventory management system.
Role-based access control for authorization.
Frequency As needed, typically for stock replenishment, redistribution, or inter-store adjustments.
Assumptions Destination location is prepared to receive and confirm receipt of transferred inventory.
Notes / Open Issues Determine if a reconciliation process is needed when items arrive at the destination for additional verification.
