User Management (CRUD Employee Permissions)

Use Case ID UM-03 Version 1.0
Use Case Name CRUD Employee Permissions Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☐ Business Requirement
☒ System Analysis
Source Business Requirement
Primary Actor Admin
Other Actor(s) None
Stakeholders
  • HR
  • IT Department
  • Management
Description This use case allows the Admin to manage individual employee permissions, assigning specific access levels as needed. It ensures that employees have the appropriate permissions for their roles within the system.
Precondition(s)
  • Admin is logged in and authenticated.
  • Employee records and permissions database are accessible.
Trigger(s) Admin initiates the process to create, view, update, or delete an employee's permissions.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Admin selects "Manage Employee Permissions" 1. System displays list of employees and permissions
2. Admin chooses an employee to assign permissions 2. System shows current permissions assigned to the employee
3. Admin selects "Add Permission" or "Remove Permission" 3. System provides available permissions for assignment or removal
4. Admin confirms and saves changes 4. System updates permissions for the employee and confirms changes

Alternate Flow

Actor Action System/Result, Info required
1. Admin attempts to assign incompatible permissions 1. System warns that certain permissions cannot coexist
Conclusion Admin successfully updates the employee's permissions, ensuring access control is tailored to their role.
Post condition Employee's permissions are saved and updated in the system database, with access changes effective immediately.
Business Rules
  • Only Admin has access to modify employee permissions.
  • Permissions should be consistent with the employee's job role and department.
Special Requirements System should display role recommendations based on the employee's department and title.
Constraints and Specifications Permissions must align with organizational policies and security requirements.
Technology and Data System database with role and permission mappings for each employee.
Frequency As needed, typically during onboarding, role changes, or employment termination.
Assumptions Admin is knowledgeable about the employee's responsibilities and access requirements.
Notes / Open Issues Consider automated notifications to the employee and relevant managers upon permissions update.
