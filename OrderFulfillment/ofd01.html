USE CASE NARRATIVE

Order Fulfillment (Fulfill Store Order)

Use Case ID OFD-01 Version 1.0
Use Case Name Fulfill Store Order Date 2024-11-02
Author Tony Power Complexity Medium
Priority High
Use Case Type:
☒ Business Requirement
☐ System Analysis
Source Business Requirement
Primary Actor Warehouse Worker
Other Actor(s) Delivery Driver
Stakeholders
  • Warehouse Operations
  • Store Managers
Description This use case describes how a warehouse worker fulfills a store order by preparing items for pickup or delivery.
Pre Condition(s) The store order has been processed and assigned items from available inventory.
The warehouse has access to the details of the store order.
Triggers Warehouse worker receives a notification to fulfill an assigned store order.

Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Warehouse Worker selects the "Fulfill Store Order" option. System displays store order details, including items and quantities to be fulfilled.
2. Warehouse Worker retrieves items from storage. System updates the order status to "In Progress."
3. Warehouse Worker scans items and packs them for shipment. System updates records and flags any discrepancies.
4. System confirms items and marks them as packed for shipping. System marks the order as "Ready for Pickup/Delivery

Alternate Flow

Actor Action System/Result, Info required
1. Warehouse Worker cannot locate an item. System prompts the worker to report a missing item and update the order status to "Pending."
Conclusion The store order is ready for pickup or delivery.
Post Condition(s) The order is marked as fulfilled and ready for delivery, and the inventory is updated.

Additional Information

Business Rules
  • Only authorized warehouse workers can access and fulfill store orders.
Special Requirements Scanners or mobile devices to confirm item retrieval.
Constraints and Specifications Must be completed by warehouse closing time.
Technology and Data Requires access to inventory management software.
Frequency As needed based on store orders.
Assumptions All ordered items are available for fulfillment.
Notes / Open Issues None
