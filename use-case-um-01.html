Basic Flow (Typical Course of Events)

Actor Action System/Result, Info required
1. Admin selects "Manage Employees" option 1. System displays list of current employees
2. Admin chooses "Add New Employee" 2. System prompts for employee details (e.g., name, role)
3. Admin enters required employee data and submits 3. System saves new employee data and confirms entry
4. Admin selects an existing employee to edit 4. System displays current data for the selected employee
5. Admin updates details and saves changes 5. System confirms changes are saved
6. Admin selects "Delete" for an employee 6. System prompts for confirmation before deletion
7. Admin confirms deletion 7. System removes employee record and confirms deletion

Additional Information

Business Rules
  • Only Admin has permission to manage employee records.
  • Mandatory fields (e.g., employee name, role) must be completed before saving.
Special Requirements System should validate employee information for accuracy.
Constraints and Specifications Database access must be secure and limited to authorized personnel.
Technology and Data System should operate on a secure server with database capabilities for data storage.
Frequency As needed, typically during onboarding, role changes, or terminations.
Assumptions Admin understands the employee management process.
Notes / Open Issues Verify if additional fields are required for employee records (e.g., emergency contact info).
